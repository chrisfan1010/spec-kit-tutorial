# /speckit.clarify

> 阶段 1.5：澄清模糊点

## 概述

`/speckit.clarify` 用于系统化地解决 spec.md 中的所有模糊点。这是一个可选但强烈推荐的阶段。

## 功能

1. **模糊性扫描**：按 9 个类别检查规格的模糊性：
   - 功能范围
   - 领域与数据模型
   - 交互与体验
   - 非功能属性
   - 集成对接
   - 边界情况
   - 约束条件
   - 术语定义
   - 完成信号

2. **状态标记**：每个类别标记为：
   - ✅ Clear（清晰）
   - ⚠️ Partial（部分清晰）
   - ❌ Missing（缺失）

3. **澄清问题生成**：生成最多 5 个优先级最高的澄清问题

4. **交互式问答**：逐个向你提问（选择题或简答题）

5. **自动更新**：将你的回答自动写入 spec.md 的对应章节

## 输出

- 更新后的 `spec.md`
- 澄清记录

## 使用方法

```bash
# 澄清所有模糊点
/speckit.clarify

# 聚焦特定领域
/speckit.clarify Focus on security and performance requirements.
```

## 9 个检查类别详解

| 类别 | 检查内容 |
|------|---------|
| 功能范围 | 需求是否完整？是否有遗漏的场景？ |
| 领域与数据模型 | 实体关系是否清晰？数据字段是否完整？ |
| 交互与体验 | UI/UX 需求是否明确？用户流程是否清晰？ |
| 非功能属性 | 性能、安全、可访问性要求是否明确？ |
| 集成对接 | 第三方 API、依赖项是否明确？ |
| 边界情况 | 异常情况、空值、并发问题是否考虑？ |
| 约束条件 | 预算、时间、技术限制是否明确？ |
| 术语定义 | 专业术语、缩写是否都有定义？ |
| 完成信号 | 怎样算完成？验收标准是否可测试？ |

## 重要提示

- 此命令会修改 spec.md 文件
- 确保没有重复、矛盾或含糊的内容残留
- 建议在 Plan 阶段之前完成
