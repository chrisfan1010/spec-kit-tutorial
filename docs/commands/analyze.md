# /speckit.analyze

> 阶段五：做检查

## 概述

`/speckit.analyze` 进行跨文件一致性检查——确保规格、计划、任务、代码四者对齐。

## 功能

1. **重复检测**：检查是否有语义重复的需求
2. **模糊检测**：检查是否有未解决的 `[NEEDS CLARIFICATION]`
3. **欠规格检测**：需求是否缺乏可衡量的成功标准
4. **宪法冲突检测**：实现是否违反宪法中的 MUST 条款
5. **覆盖缺口检测**：是否有需求没有对应的任务
6. **术语一致性**：不同文档中是否对同一概念使用了不同名称

## 输出

- 结构化分析报告（最多 50 条发现）
- 每个发现包含：严重级别、具体位置、修复建议

## 严重级别

| 级别 | 说明 |
|------|------|
| CRITICAL | 必须修复（宪法冲突等） |
| HIGH | 强烈建议修复 |
| MEDIUM | 建议修复 |
| LOW | 可选修复 |

## 检查项详解

| 检查项 | 说明 | 严重级别 |
|--------|------|---------|
| 重复检测 | 是否有语义重复的需求 | MEDIUM |
| 模糊检测 | 是否有未解决的 `[NEEDS CLARIFICATION]` | HIGH |
| 欠规格 | 需求是否缺乏可衡量的成功标准 | HIGH |
| 宪法冲突 | 实现是否违反宪法中的 MUST 条款 | **CRITICAL** |
| 覆盖缺口 | 是否有需求没有对应的任务 | HIGH |
| 术语不一致 | 不同文档中是否对同一概念使用了不同名称 | MEDIUM |

## 使用方法

```markdown
/speckit.analyze
```

## 重要特性

- 此命令是**只读**的，不会修改任何文件
- 只输出检查报告，不执行修复
- 建议在 Implement 阶段完成后、提 MR 前运行
- 分析结果用于指导修复工作

## 报告示例

```markdown
## 分析报告

### 发现 1
- **级别**: HIGH
- **类型**: 模糊检测
- **位置**: spec.md line 25
- **描述**: 发现 [NEEDS CLARIFICATION] 标记未解决
- **修复建议**: 使用 /speckit.clarify 澄清此问题

### 发现 2
- **级别**: CRITICAL
- **类型**: 宪法冲突
- **位置**: src/service/UserService.java
- **描述**: 代码违反了宪法中"所有接口必须经过权限校验"的规则
- **修复建议**: 添加权限校验逻辑
```
